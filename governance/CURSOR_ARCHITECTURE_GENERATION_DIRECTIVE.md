**Version:** 1.0
**Effective Date:** January 2026
**Predecessor:** CURSOR_VALIDATION_INSTRUCTIONS.md (Phase 3)
**Successor:** CURSOR_DEVELOPMENT_GOVERNANCE.md (Phase 5)

## EXECUTIVE SUMMARY

Before Cursor writes a single line of production code, this directive mandates the creation of two exhaustive flowchart artifacts that capture the complete system architecture. These flowcharts must be comprehensive enough that:

1. A competent developer unfamiliar with CIA-SIE could implement the complete system from them alone
2. A program analyst could verify complete coverage with zero gaps
3. Every state transition is explicit—no unhandled operations possible
4. Constitutional compliance is architecturally enforced, not bolted on

## PART I: DIRECTIVE PHILOSOPHY

### 1.1 Why Architecture-First

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      ARCHITECTURE-FIRST PRINCIPLE                            │
│                                                                              │
│   Code without architecture = Technical debt accumulation                   │
│   Architecture without code = Clear implementation path                     │
│                                                                              │
│   The flowcharts ARE the specification.                                     │
│   Development becomes "implement the flowchart."                            │
│   Deviation from flowchart = Violation requiring justification.             │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 Flowchart Standards

All flowcharts must:

| Requirement | Description |
|-------------|-------------|
| **Exhaustive** | Cover every component, state, and transition |
| **Explicit** | No implied behavior—everything stated |
| **Testable** | Each flow verifiable via test |
| **Constitutional** | Rules enforced architecturally |
| **Mermaid Format** | Use Mermaid.js for rendering |
