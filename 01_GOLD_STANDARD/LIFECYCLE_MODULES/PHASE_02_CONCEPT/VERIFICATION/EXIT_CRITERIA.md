# PHASE 02: CONCEPT CRYSTALLIZATION
## Exit Criteria

**Purpose:** Conditions that MUST be satisfied before proceeding to Phase 3

---

## MANDATORY DELIVERABLES

- [ ] **Concept Document Exists**
  - Detailed description of what will be built
  - Target users identified
  - Key capabilities listed
  - Understandable by all stakeholders

- [ ] **Scope Definition Complete**
  - In-scope items explicitly listed
  - Out-of-scope items explicitly listed
  - Deferred items documented

- [ ] **Components Identified**
  - Major system components listed
  - High-level relationships understood
  - No detailed design required

- [ ] **Risks Registered**
  - Known risks documented
  - Impact and likelihood assessed
  - Mitigation strategies identified

- [ ] **Success Criteria Defined**
  - Measurable outcomes listed
  - Not vague ("fast", "good")
  - Verifiable at project end

---

## QUALITY CHECKS

- [ ] **Completeness**
  - All sections of concept document filled
  - No TBD or placeholder content

- [ ] **Clarity**
  - Non-technical stakeholder can understand
  - No jargon without definition

- [ ] **Consistency**
  - Aligns with Phase 1 idea statement
  - No internal contradictions

- [ ] **Stakeholder Alignment**
  - Principal confirms concept matches intent
  - Key stakeholders have reviewed
