# PHASE 04: ARCHITECTURE DESIGN
## Exit Criteria

**Purpose:** Conditions that MUST be satisfied before proceeding to Phase 5

---

## MANDATORY DELIVERABLES

- [ ] **Context Diagram (C4 L1) Complete**
  - System boundary clearly defined
  - All external actors identified
  - All external systems identified
  - Relationships documented

- [ ] **Container Diagram (C4 L2) Complete**
  - All containers (applications, services, databases) identified
  - Container responsibilities defined
  - Inter-container communication documented
  - Technology choices indicated

- [ ] **Architectural Decision Records**
  - ADR for each significant decision
  - Status marked (Proposed/Accepted)
  - Context clearly explained
  - Alternatives documented

- [ ] **Quality Attribute Scenarios Defined**
  - Performance scenarios documented
  - Scalability approach defined
  - Security architecture outlined
  - Maintainability considerations addressed

---

## QUALITY CHECKS

- [ ] **Completeness**
  - All requirements addressable by architecture
  - No orphan requirements
  - All quality attributes covered

- [ ] **Consistency**
  - No contradictory decisions
  - Patterns applied consistently
  - Terminology consistent

- [ ] **Feasibility**
  - Architecture achievable with resources
  - Technology choices viable
  - Team has necessary skills (or plan to acquire)

- [ ] **Traceability**
  - Architecture traces to requirements
  - Decisions trace to drivers (requirements, constraints)

- [ ] **Reviewability**
  - Diagrams clear and understandable
  - ADRs readable by stakeholders
  - No unexplained jargon

---

## RISK ASSESSMENT

- [ ] **Risks Identified**
  - Technical risks documented
  - Integration risks assessed
  - Skill gap risks noted

- [ ] **Mitigations Planned**
  - Each risk has mitigation strategy
  - Contingencies documented
  - Risk owners assigned
