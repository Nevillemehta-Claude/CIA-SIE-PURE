## PURPOSE

This module contains universal standards for documentation - what to document, how to document it, how to keep it current, and how to make it useful. Good documentation accelerates development; bad documentation impedes it.

---

## SCOPE

### What This Module Covers

- Documentation philosophy and principles
- Types of documentation and their purposes
- Living documentation practices
- Code documentation standards
- API documentation standards
- User documentation standards
- Documentation quality metrics

### What This Module Does NOT Cover

- Project-specific document templates (see PROJECT_CHRONICLES)
- Tool-specific formatting (reference tool docs)
- Content of specific documents
